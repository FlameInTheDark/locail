# Repository Guidelines

## Project Structure & Module Organization
- Go backend (Wails): root `main.go`, `app.go`; config `wails.json`.
- Frontend (React + TypeScript + Vite): `frontend/` (`src/main.tsx`, `src/App.tsx`, assets in `src/assets/`).
- Generated bindings: `frontend/wailsjs/**` (regenerated by Wails; do not edit).
- Build artifacts: `build/**`; production assets in `frontend/dist` embedded via `go:embed`.

## Build, Test, and Development Commands
- Install frontend deps (first run): `cd frontend && npm install`.
- Live development (Go + frontend): `wails dev`.
- Frontend only dev server: `cd frontend && npm run dev`.
- Frontend build + type-check: `cd frontend && npm run build`.
- Production desktop build: `wails build` (binary in `build/bin/locail`).
- Quick run of built app: `./build/bin/locail`.

Prereqs: Go 1.23+, Node 16+ (or newer LTS), Wails CLI: `go install github.com/wailsapp/wails/v2/cmd/wails@latest`.

## Coding Style & Naming Conventions
- Go: `gofmt -s -w .` and `go vet ./...` must pass. Packages lowercase; exported names PascalCase; internal names camelCase.
- TS/React: 2-space indent; components PascalCase (`SettingsPanel.tsx`); hooks `useX`. Prefer function components and ES modules. Colocate styles (`.css`) next to components.
- Never edit `frontend/wailsjs/**`; regenerate via `wails dev` or `wails build`.

## Testing Guidelines
- Go tests: `*_test.go` adjacent to sources; run `go test ./...`. Favor small, focused units around exported behavior.
- Frontend tests: not configured. If adding, use `vitest` + `@testing-library/react` with `*.test.tsx` under `frontend/src/`.

## Commit & Pull Request Guidelines
- Conventional Commits: `feat:`, `fix:`, `chore:`, `docs:`, `refactor:`, etc., with optional scope (e.g., `feat(frontend): add settings view`).
- Keep commits small, reference issues (`#123`), and update README when user-visible behavior changes.
- PRs must include description, motivation, screenshots/GIFs for UI, and validation steps. Ensure formatting, `go vet`, and `wails build` succeed.

## Security & Configuration Tips
- Do not commit secrets. Review `wails.json` hooks before changing build steps.
- Place images/fonts in `frontend/src/assets/` so Vite bundles them into `frontend/dist` for embedding.
